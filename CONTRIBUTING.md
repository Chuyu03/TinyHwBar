# Contributing to TinyHwBar

感谢你考虑改进 TinyHwBar。请保持变更小而清晰，并优先保护本项目的低资源占用、本地运行和隐私边界。

## 开发环境

- Windows 11 x64。
- .NET Framework 4.8 自带的 64 位 C# 编译器。
- 不需要 .NET SDK、NuGet 或第三方包。

在项目根目录运行：

```powershell
.\build.cmd
```

构建必须保持零警告、零错误，并生成 `outputs\TinyHwBar.exe`。

## 提交变更

1. Fork 仓库并从 `main` 创建短小、单一目的的分支。
2. 不要提交 `outputs/`、本地设置、日志、凭据或个人路径。
3. 不要引入网络请求、遥测、远程接口、内核驱动或额外依赖，除非维护者事先明确接受相应范围和风险。
4. 对硬件采样、P/Invoke、构建脚本或工作流的修改必须说明安全影响和失败回退。
5. 在 PR 中列出实际运行的构建与手动验证；不要把“可以编译”描述成“代码安全”。

外部贡献者的自动构建需要维护者先审查并批准。任何工作流或构建脚本改动都会被视为高风险范围。

## 报告安全问题

不要在公开 Issue 中披露可利用细节、凭据或敏感日志。请按 [SECURITY.md](SECURITY.md) 使用 GitHub 私密漏洞报告。
