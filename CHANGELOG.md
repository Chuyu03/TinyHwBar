# Changelog

TinyHwBar 的重要版本变化记录在此文件中。版本号遵循语义化版本格式。

## [Unreleased]

## [2.0.0] - 2026-07-16

### Added

- 选定并生成正式的极简仪表盘指针图标，提供 16–256 像素多尺寸 ICO 和透明 PNG 源图。
- 新增 `50%`–`100%` 透明度选项，并保存用户选择。
- V2 新增四页控制中心：概览、历史曲线、设置和高级功能。
- 新增明确的键盘导航顺序、关键控件辅助说明和浅色曲线对比度检查。
- 新增基于 Windows 路由表的网络适配器选择、被动上下行吞吐和接口标称链路速率；支持路由指向的 VPN Tunnel 与 IPv6-only 环境，并对 IPv4/IPv6 分流明确降级而不猜选接口或执行带宽测试。
- 新增所选路由接口报告的本地网关 ICMP 延迟：默认开启，每 10 秒一次、750 毫秒超时，仅允许 RFC1918、CGNAT、link-local 和 ULA 目标；公网/全局目标拒绝。多网关时不把稳定选择结果误称为路由 next hop。
- GPU 概览改为“独立显卡 / 核显”角色分类：枚举 NVIDIA、AMD、Intel 物理适配器，在可可靠识别时排除远程、软件和虚拟候选，按 LUID 去重并通过 DXCore `IsIntegrated` 分类；Windows GPU 计数器提供角色利用率与内存，NVIDIA NVML 仅可选增强 NVIDIA 独立显卡温度。不支持、类型未知或候选不唯一时明确降级。
- 新增本地历史持久化：只保存数值型采样，内存与磁盘均最多 900 点、24 小时，文件上限 2 MiB，支持清除、原子写入和一次性失败提示。
- 新增默认关闭的高级功能：当前用户启动项、更新检查、回环 API 和逐次确认的遥测/诊断摘要。
- 新增更新候选暂存：每次确认后最多下载 32 MiB，验证清单 SHA-256，只暂存而不执行、安装或替换当前程序。
- 新增仅 `127.0.0.1` 的随机高端口 API，固定 `/v1/status`、每次运行 token 和有界数值响应。
- 新增当前用户安装/卸载脚本；支持 `-WhatIf`，默认安装不启动且不启用开机启动，默认卸载保留用户数据。
- 新增本地边界测试和仅编译测试模式。

### Changed

- 监控条可在任意显示器完整边界内自由拖动并恢复位置，包括任务栏预留但视觉上空闲的区域；“重置到右上角”仍使用主屏工作区作为安全落点。
- 控制中心显卡卡片调整为“核心指标优先”的信息层级：利用率、显存/共享内存和可用温度保持大字号单行完整显示，设备型号在次行以较小字号显示，空间不足时仅缩略型号。
- 设置与历史使用临时文件加原子替换，写入失败时尽量保留旧文件。
- GPU 历史曲线改用“独立显卡 / 核显”名称；磁盘历史架构和列顺序保持兼容，角色命名变更不会主动清空已有记录，后续正常持久化仍沿用原子保存流程。
- 本地状态 API 响应升级为 `schemaVersion: 2` 的 `discrete* / integrated*` GPU 字段；旧 `nvidia* / intel*` 数值字段只在实际厂商匹配时作为兼容别名返回。
- 正式版本号更新为 `2.0.0`；产品、进程、窗口和 EXE 名称仍保持为 `TinyHwBar`。

### Fixed

- 修复设备型号挤占显卡卡片主区域、导致利用率、显存或温度等核心数值被遮挡或省略的问题。
- 修复同厂商、同型号但 LUID 不同的真实 GPU 可能被当作同一适配器镜像折叠的问题；只有厂商、型号和完整 LUID 均一致的未知候选才按镜像去重，其余情况明确降级为歧义而不猜选。
- 将临时测试程序名统一为 `TinyHwBar.Tests.*.exe`，避免 Windows 安全通知继续显示旧的 V2 开发后缀。
- 修复映射网络盘可绕过开机启动“仅本地文件系统路径”校验的问题；无法确认卷类型时现在也会安全拒绝。
- 测试脚本会避开既有临时 EXE 名称，并在清理失败时明确报错，避免静默遗留未签名测试程序。

### Known limitations

- 同一角色存在多个物理 GPU 时，当前版本不会自动选择其中一张，而是明确显示歧义状态。
- 顶部监控条保持 `330×36` 物理像素；Per-Monitor-V2 基础设施已启用，但不同缩放比例的多显示器组合仍缺少系统性人工验收。
- 可选回环 API 使用单一后台工作线程；本机慢连接可使 API 单次暂时不可用至约 5 秒，但不会阻塞监控采样或界面。
- 当前构建未签名，Smart App Control 仍可能阻止执行。

### Security

- HTTPS 高级功能端点默认为空，并拒绝 query、userinfo 和 fragment；启动时自动更新、API 与遥测均保持默认关闭。
- V1 设置迁移只保留 V1 已定义字段；即使旧文件混入同名字段，V2 的启动项、更新、API、遥测和端点仍使用安全默认值。
- 明确 SHA-256 清单校验不等于发布者身份或代码签名。
- 本机复现了 Smart App Control 对未签名、未建立云信任的临时测试 EXE 的阻止；SignPath Foundation 仍仅完成评估，没有申请、接入或签名。

## [1.0.0] - 2026-07-12

### Added

- `330×36` 物理像素的单行 Windows 硬件监控条。
- CPU、物理内存、NVIDIA GPU 占用、显存比例和温度采样。
- 电池模式 `GPU ECO`，停止 NVML 调用以避免主动保持独显唤醒。
- 拖动、位置锁定、鼠标穿透、托盘显示/隐藏和位置恢复。
- Per-Monitor-V2 DPI、单实例保护和本地 INI 设置持久化。
- 无依赖的 .NET Framework 4.8 x64 构建脚本。
- 只读、无密钥且不保存产物的 GitHub Actions 编译检查。
- 强化 NVIDIA NVML 动态库加载边界。
